# Memory Techniques

## What It Is

The deliberate use of structure to extend the reach of human memory beyond its natural limits. Not rote repetition — the exploitation of how memory actually works: spatially, associatively, visually, narratively. These techniques are ancient (the Greeks knew them), powerful (competitive memorisers use them), and almost nobody in ordinary life bothers to learn them. They turn memory from a passive capacity into an active skill.

---

## Core Concepts

### How Memory Works

- **Encoding, storage, retrieval** — three stages, each with its own failure modes. Most "forgetting" is a retrieval failure, not a storage one. The information is there; the path to it is lost.
- **Working memory** — the bottleneck. Holds 4±1 chunks at a time (Miller's original 7±2 was for digits; chunked information reduces the count). Everything that enters long-term memory passes through this narrow gate.
- **Chunking** — grouping items into meaningful units to bypass working memory limits. "FBI CIA NSA" is three chunks, not nine letters. Expertise IS chunking — a chess grandmaster sees board positions, not individual pieces.
- **Dual coding** — information encoded both verbally and visually is remembered far better than either alone. This is why the best memory techniques are intensely visual.
- **Elaborative encoding** — the more connections you make to existing knowledge, the stronger the memory. Shallow processing (what does the word look like?) vs deep processing (what does it mean? how does it connect?).
- **Spacing effect** — distributed practice beats massed practice. Reviewing something at increasing intervals produces stronger retention than cramming. This is the basis of spaced repetition systems.
- **Testing effect** — actively retrieving information strengthens memory more than passively reviewing it. The act of recall is itself the training.
- **Interference** — similar memories compete. Proactive interference (old memories block new) and retroactive interference (new memories blur old). Distinctiveness protects against interference.
- **Context-dependent memory** — memory is better when retrieval context matches encoding context.
- **State-dependent memory** — emotional and physiological state at encoding affects retrieval. Fear-state memories are best recalled in fear states.
- **Reconstruction** — memory is not playback. Every act of recall is a reconstruction, subject to distortion, confabulation, and reinterpretation. Memories change each time they are accessed.

### The Method of Loci (Memory Palace)

The foundational technique, attributed to Simonides of Ceos (5th century BC). Cicero describes it in *De Oratore*. The method:

1. Choose a familiar spatial environment (a building, a route, a room)
2. Identify a sequence of distinct locations (loci) within that environment
3. Mentally place vivid, bizarre, interactive images at each location
4. To recall, mentally walk the route and "see" what's at each location

**Why it works:** Human spatial memory is extraordinarily powerful — we evolved to navigate complex environments. The method of loci exploits this by converting abstract information into spatial-visual form. The loci provide a stable retrieval structure; the images provide distinctive, elaborately encoded content.

**Capacity:** A single well-known building can hold hundreds of loci. Competitive memorisers maintain dozens of palaces with thousands of loci total. There is no known upper limit.

### The Dominic System

Developed by Dominic O'Brien (eight-time World Memory Champion). A person-action system for encoding numbers.

**The mapping:**
- Each two-digit number (00–99) maps to a person via letter pairs
- The digits map to letters: 1=A, 2=B, 3=C, 4=D, 5=E, 6=S, 7=G, 8=H, 9=N, 0=O
- The two letters become someone's initials: 15 = AE = Albert Einstein. 67 = SG = Steffi Graf.
- Each person has a characteristic action: Einstein writes on a blackboard. Graf serves a tennis ball.

**How it's used:**
- To memorise a long number, break it into two-digit pairs
- Convert each pair to a person
- For four-digit groups, combine: the person from the first pair performs the action of the second pair
- Place these person-action images at loci in a memory palace

**Example:** To remember 1567: 15 = Albert Einstein, 67 = Steffi Graf. Image: Einstein serving a tennis ball. Place at a locus.

### PAO (Person-Action-Object)

An extension of the Dominic system used by most competitive memorisers.

- Each two-digit number maps to a Person, an Action, and an Object
- For six-digit groups: take the Person from the first pair, the Action from the second, the Object from the third
- This creates a single composite image per six digits

**Example:** If 13 = Clint Eastwood (person: cowboy, action: shooting, object: revolver), 25 = Brad Pitt (person: actor, action: punching, object: soap), 74 = Gareth Bale (person: footballer, action: kicking, object: football):
- 132574 = Clint Eastwood kicking a football
- One image. Six digits. Placed at one locus.

**Density:** PAO encodes information three times more densely than simple person systems. A 50-locus palace can hold 300 digits.

### The Major System (Phonetic Number System)

Older than the Dominic system. Maps digits to consonant sounds:

- 0 = s, z (zero starts with z)
- 1 = t, d (one downstroke)
- 2 = n (two downstrokes)
- 3 = m (three downstrokes)
- 4 = r (four ends with r)
- 5 = l (L is the Roman numeral for 50)
- 6 = j, sh, ch (J looks like a mirror-6)
- 7 = k, g (K looks like two 7s)
- 8 = f, v (cursive f looks like 8)
- 9 = p, b (p is a mirror-9)

Vowels are free — fill them in to make words. So 42 = "rain" (r-n). 95 = "bell" (b-l). 7215 = "candle" (k-n-d-l).

**Advantage over Dominic:** Generates words directly, which can be more flexible. Disadvantage: requires more mental computation.

### Peg Systems

Pre-memorised lists that provide instant hooks for new information.

**Number-shape pegs:** 1 = candle (looks like 1), 2 = swan (looks like 2), 3 = trident, 4 = yacht (sail), 5 = hook, etc. To remember a list, link each item to its peg via a vivid image.

**Number-rhyme pegs:** 1 = gun, 2 = shoe, 3 = tree, 4 = door, 5 = hive, 6 = sticks, 7 = heaven, 8 = gate, 9 = wine, 10 = hen.

**Why pegs matter:** They provide O(1) access — you can go directly to item 7 without traversing items 1–6. Memory palaces are sequential; peg systems are random-access.

### The Link System (Story Method)

The simplest technique. Chain items together in a narrative sequence, each linked to the next by a vivid image or interaction.

To remember: apple, piano, elephant, moon — imagine an apple crashing onto piano keys, the sound startling an elephant who trumpets at the moon.

**Strengths:** No setup required, works for any list, easy to learn. **Weaknesses:** Sequential access only, fragile (break one link, lose the rest), no good for large sets.

### Spaced Repetition

Not a memory technique per se but a scheduling algorithm for review.

- New information is reviewed at increasing intervals: 1 day, 3 days, 7 days, 14 days, 30 days...
- Items recalled correctly get longer intervals; items forgotten get shorter ones
- Based on Ebbinghaus's forgetting curve (1885) and Pimsleur's graduated interval recall
- Modern implementations: Anki, SuperMemo, Leitner box system

### Mnemonics and Encoding Strategies

- **Acronyms and acrostics** — "Roy G. Biv" for rainbow colours. Compresses a list into a word.
- **Rhyme and rhythm** — "Thirty days hath September..." Melody and metre are powerful encoding aids. The litanies may work partly because their rhythm makes them memorable.
- **Visual association** — the more bizarre, vivid, and emotionally charged the image, the better it's remembered. Memory champions deliberately create absurd, violent, sexual, or impossible images.
- **The von Restorff effect** — distinctive items in a series are better remembered than common ones. The one red card in a deck of black cards.
- **Narrative encoding** — information embedded in a story is better remembered than information in a list. The story provides causal links between items.

## Key Thinkers

- **Simonides of Ceos** — traditional inventor of the method of loci (5th century BC). Identified the bodies of banquet guests crushed by a roof collapse by remembering where each sat.
- **Cicero** — described the method of loci in *De Oratore*. Recommended vivid, striking images.
- **Giordano Bruno** — Renaissance memory artist. Used elaborate memory palaces incorporating astrological and hermetic imagery. Burned at the stake.
- **Matteo Ricci** — Jesuit missionary who taught the method of loci to Chinese scholars (16th century). Wrote *A Treatise on Mnemonic Arts*.
- **Hermann Ebbinghaus** — discovered the forgetting curve and the spacing effect (1885). Showed that memory decay is exponential but can be fought with timed review.
- **Dominic O'Brien** — eight-time World Memory Champion. Developed the Dominic System. Demonstrated that memory is a trainable skill, not a fixed capacity.
- **Joshua Foer** — journalist who trained for a year and won the US Memory Championship. Wrote *Moonwalking with Einstein*. Showed that ordinary people can develop extraordinary memory.
- **Frances Yates** — historian who traced the art of memory from Greece through the Renaissance in *The Art of Memory* (1966). Showed memory techniques as a continuous Western tradition.
- **Tony Buzan** — popularised mind mapping and memory techniques. Founder of the World Memory Championships.
- **Piotr Wozniak** — creator of SuperMemo, the first spaced repetition software. Developed the SM-2 algorithm.
- **Sebastian Leitner** — invented the Leitner box system for spaced repetition using physical flashcards.

## Connections

- **Epistemology** — memory is a source of knowledge, but an unreliable one. Memory-based beliefs require the same scrutiny as observation-based ones. The testing effect suggests that retrieval IS a form of knowledge construction.
- **Logic** — memory techniques impose logical structure (sequence, categorisation, hierarchy) on unstructured information. The method of loci IS a logical system — a map from positions to propositions.
- **Probability** — calibrating confidence in memories. "I think I remember X" is a probabilistic claim. How reliable is it? Base rates for memory accuracy under different conditions (stress, delay, interference) matter.
- **Emotional intelligence** — state-dependent memory means emotional state affects recall. Fear-state learning produces fear-state memories. Regulating emotion is partly about managing what you can access in memory.
- **Decision-making** — decisions under time pressure depend on what you can retrieve quickly. Memory techniques that provide rapid access (pegs, PAO) directly improve decision quality. The availability heuristic means poorly remembered options are underweighted.
- **Algebra** — the Dominic system is literally a mapping function: digits → letters → people → actions. PAO is a function composition. Memory techniques ARE algebraic structures, even when the user doesn't think of them that way.
- **Stoicism** — Marcus Aurelius used journaling (the *Meditations*) as an external memory system for principles. The litanies serve the same function. Premeditatio malorum is a form of prospective memory — encoding future responses to anticipated situations.
- **Rationalism** — the sequences on LessWrong function as a curriculum explicitly designed for memory: ideas are introduced, repeated in new contexts, and tested through application.
